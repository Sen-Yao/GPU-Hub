# GPUHub 架构设计

> **版本**: v1.0
> **设计原则**: 重总控端，轻算力端

---

## 一、整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      Client Layer                            │
│  (OpenClaw / 外部应用 / 用户) → HTTP Request                 │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│              总控端 - Yggdrasil (Docker)                      │
│  ┌─────────────┬─────────────┬─────────────┬─────────────┐  │
│  │ FastAPI     │ Redis Queue │ MySQL DB    │ 前端仪表盘  │  │
│  │ (调度)      │ (队列)      │ (请求跟踪)  │ (监控)      │  │
│  └─────────────┴─────────────┴─────────────┴─────────────┘  │
│  端口: 8001 | 外网: Cloudflare Tunnel                       │
└─────────────────────────────────────────────────────────────┘
                              ↓ (HTTP/WebSocket 心跳)
┌─────────────────────────────────────────────────────────────┐
│              算力端 - HCCS86 (Python Worker)                 │
│  ┌─────────────┬─────────────┬─────────────┐               │
│  │ Node Agent  │ Executors   │ GPU Monitor │               │
│  │ (心跳上报)  │ (llama/whisper)│ (nvidia-smi)│               │
│  └─────────────┴─────────────┴─────────────┘               │
│  8 × NVIDIA L40 (48GB each)                                 │
└─────────────────────────────────────────────────────────────┘
```

---

## 二、核心原则

| 原则 | 说明 |
|------|------|
| **重总控端，轻算力端** | 总控端负责调度、队列、权限、记录；算力端只执行 |
| **统一入口** | 客户端只与总控端交互，无需知道后端拓扑 |
| **扩展性** | 支持多服务器接入，动态扩容 |
| **可观测性** | 前端仪表盘 + 完整执行记录 |

---

## 三、技术栈

| 模块 | 技术栈 |
|------|--------|
| **总控端后端** | Python FastAPI |
| **任务队列** | Redis |
| **请求跟踪** | MySQL |
| **前端仪表盘** | HTML + Tailwind CSS |
| **算力端代理** | Python Worker |
| **通信协议** | HTTP/WebSocket |
| **外网访问** | Cloudflare Tunnel |

---

## 四、数据流

### 4.1 请求处理流程

```
Client → Control Plane (接收)
              ↓ (验证)
              ↓ (存入 MySQL)
              ↓ (加入 Redis 队列)
              ↓ (调度器选择节点)
              ↓ (分发到 Node Agent)
Node Agent → 执行任务
              ↓ (返回结果)
Control Plane → 更新 MySQL
              ↓ (返回 Client)
```

### 4.2 心跳上报流程

```
Node Agent (每10秒) → Control Plane /heartbeat
              ↓ (更新 Redis 缓存)
              ↓ (记录节点状态)
              ↓ (前端仪表盘读取)
```

## 五、中文术语

| 英文 | 中文 | 说明 |
|------|------|------|
| **Control Plane** | 总控端 | Yggdrasil 上的调度中心 |
| **Worker Node** | 算力端 | HCCS86 等执行节点 |
| **Node Agent** | 算力端代理 | 部署在算力端的服务进程 |
| **Executor** | 执行器 | llama / whisper 等具体程序 |
