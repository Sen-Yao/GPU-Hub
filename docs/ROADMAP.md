# GPUHub Roadmap

> **版本演进规划**
> **当前版本**: V1

---

## V1 核心功能（当前）

| 功能模块 | 说明 | 状态 |
|---------|------|------|
| **单节点接入** | 单个 Worker Node | ✅ 当前 |
| **总控端** | Control Plane Docker + Redis + MySQL | ✅ 当前 |
| **算力端代理** | Python Worker | ✅ 当前 |
| **请求跟踪** | 状态流转 + MySQL 存储 | ✅ 当前 |
| **队列维护** | Redis 任务队列 | ✅ 当前 |
| **基础调度** | FIFO + GPU 显存判断 | ✅ 当前 |
| **前端仪表盘** | Requests + Nodes + Queues | ✅ 当前 |
| **心跳机制** | HTTP/WebSocket 轮询 | ✅ 当前 |
| **任务类型** | chat / embedding / stt | ✅ 当前 |

---

## V2 功能扩展（规划）

| 功能模块 | 说明 | 状态 |
|---------|------|------|
| **多节点接入** | 支持第 2+ 个算力端 | ⏳ V2 |
| **节点能力标签** | `supports: [llm, stt]` + `gpu_vendor` | ⏳ V2 |
| **细粒度 GPU 分配** | 单任务多卡、显存预留 | ⏳ V2 |
| **故障转移** | 节点离线时任务重调度 | ⏳ V2 |
| **优先级队列** | 高/中/低优先级任务 | ⏳ V2 |
| **配额管理** | 用户级配额限制 | ⏳ V2 |
| **Tailscale 集成** | overlay 网络替代 HTTP 轮询 | ⏳ V2 |
| **更强调度策略** | 负载均衡 + 资源亲和性 | ⏳ V2 |

---

## V3 功能扩展（规划）

| 功能模块 | 说明 | 状态 |
|---------|------|------|
| **分片执行** | 单任务跨多节点 | ⏳ V3 |
| **多节点协同** | 分布式推理（如 llama 分片） | ⏳ V3 |
| **插件化 Executors** | 动态加载新任务类型 | ⏳ V3 |
| **自动扩缩容** | 动态添加/移除节点 | ⏳ V3 |
| **审计日志** | 操作审计 + 异常追溯 | ⏳ V3 |
| **API Gateway** | 对外提供统一 API 入口 | ⏳ V3 |

---

## 网络通信演进

| 版本 | 通信方式 | 说明 |
|------|---------|------|
| **V1** | HTTP/WebSocket 轮询 | 算力端主动连接，最小依赖 |
| **V2** | Tailscale overlay | 双方安装 tailscale，稳定低延迟 |
| **V3** | 混合模式 | WebSocket 实时 + Tailscale 备用 |

---

## 前端演进

| 版本 | 页面 | 说明 |
|------|------|------|
| **V1** | Requests / Nodes / Queues | 基础运维控制台 |
| **V2** | GPU Detail / Task History / 配额管理 | 更细粒度管理 |
| **V3** | 多节点拓扑 / 分片执行可视化 / 插件管理 | 高级运维 |
